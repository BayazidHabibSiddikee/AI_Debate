# AI Debate & Research Playground

[![Python 3.8+](https://img.shields.io/badge/python-3.8+-blue.svg)]()
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

A multi-agent chat playground designed to generate synthetic conversation datasets. 

## Mission
Our goal is to simulate deep, multi-character debates to brainstorm better ideas through AI-driven discussion and research. The exported datasets (`storage/debates_dataset.jsonl`) generated by this playground can be used to train action mappers (mapping nuanced texts to complex physical animations) for VRM models.

## Project Architecture
This project is cleanly split into two distinct processes:

1. **The Debate Simulator** (`app.py` on Port 5050): The core Flask and LangChain application that orchestrates the AI debates and exports the conversational turns.
2. **The Research & RAG Engine** (`rag_server.py` on Port 5080): Provides RAG (Retrieval-Augmented Generation) processing, news harvesting, and document context. The debate simulator optionally connects to this to ground characters with real research data.

### Directory Structure
* `templates/`: HTML files (`index.html`, `chat.html`) for the debate UI.
* `images/`: Where character PNGs with hidden metadata are stored.
* `storage/`: Auto-generated directory for SQLite DBs, ChromaDB indices, and `debates_dataset.jsonl`.
* `tools/`: Independent research scripts (e.g., `news_harvester.py`, `knowledge_hub.py`, `stealth_browser.py`).
* `utils/`: Core agent helpers (`agent_logic.py`, `platform.py`).
* `rag/`: Document chunking and embedding modules (`loader.py`, `clean_docs.py`).

## 1. Setting up the Debate Simulator

### Prerequisites
* **Python 3.8+**
* An LLM provider (Ollama, LM Studio, or OpenAI API)

### Installation
The simulator uses a lightweight dependency list.
```bash
pip install -r requirements-debate.txt
```

### Environment Setup (`.env`)
Create a `.env` file from the included `.env.example`:
```env
OPENAI_API_BASE=http://localhost:1234/v1
OPENAI_API_KEY=not-needed
LLM_MODEL=local-model
FLASK_DEBUG=false
TELEGRAM_TOKEN=your-token
TELEGRAM_CHAT_ID=your-chat-id
```

### Running the Simulator
```bash
python app.py
```
Open `http://localhost:5050` in your browser. Every debate turn is automatically appended to `storage/debates_dataset.jsonl` for your future training pipelines.

## 2. Setting up the RAG & Research Engine

### Installation
If you wish to run the background research tools, install the heavier dependency stack:
```bash
pip install -r requirements-rag.txt
```

### Important Notes for RAG/News
* **Embedding Model**: On first run, the `rag_server.py` will automatically download `all-MiniLM-L6-v2` (~90MB).
* **Ollama for News**: The `tools/news_harvester.py` module uses `ollama` natively. Ensure Ollama is installed and running if you want to use the automated news crawler.
* **Database**: `database.py` manages RAG history and user interactions. The schema is automatically initialized when the system runs.
* **Configuration**: Check `config.py` for global settings like the embedding model. Some deep agent logic also relies on `settings.json`.

### Running the RAG Server
```bash
python rag_server.py
```
The RAG server runs on **Port 5080**. If running, the Debate Simulator (`app.py`) will automatically detect it and inject relevant context into the characters' system prompts before they speak.

## Data Formats

### 1. Dataset Export (`storage/debates_dataset.jsonl`)
Every turn in the debate UI appends a JSON object to this file.
```json
{"topic": "Global Warming", "speaker": "Shikamaru Nara", "text": "What a drag. Let's look at the climate data..."}
```

### 2. Character Cards (PNG Metadata)
Character images in the `images/` folder contain embedded base64 `chara` chunks (or raw JSON in `iTXt`). 
Drop standard Character PNGs (TavernAI/SillyTavern format) into the folder, then run:
```bash
python extract_chunks.py
```
This extracts the hidden personality metadata and updates `characters.json`.

## API Endpoints

### Debate Simulator (`app.py` - Port 5050)
* `GET /`: Renders the Landing Page (`index.html`).
* `GET /chat`: Renders the Debate Simulator UI (`chat.html`).
* `GET /api/characters`: Returns all parsed characters from `characters.json`.
* `POST /api/chat`: Takes `topic`, `characters` (list), and `history` (list) to generate the next LLM response and saves it to the dataset.
* `GET /images/<path>`: Serves the avatar PNG files.

### RAG Server (`rag_server.py` - Port 5080)
* Includes endpoints for RAG context ingestion and retrieval (e.g., `/context`).
